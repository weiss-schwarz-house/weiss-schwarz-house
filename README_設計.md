# サイト要件定義（ヴァイスシュヴァルツの館）

## 1. 目的
- サークル内でプレイするボードゲームの**ルールブック・要点・注意点を素早く参照できる**ようにする
- ルール確認・勘違い防止・FAQ確認を主目的とする
- 将来的に多数（数十〜100以上）のゲーム追加を想定する
- 個人ブログではなく、**共有ナレッジ（簡易Wiki）**として運用する

---

## 2. 想定利用者
- サークルメンバー
- ボードゲーム経験者（ライト〜重量級まで）
- ルールブック全文ではなく「要点・確認」を求める利用者

---

## 3. 想定コンテンツ
- 全体トップページ
- ゲーム一覧ページ
- ゲーム個別ページ（1ゲーム1ページ）
- 外部ルールPDFへのリンク
- ゲームごとの要点・勘違いやすい点・気づき・Q&A

---

## 4. ページ構成・ディレクトリ構成
/
├ index.md # 全体トップ
├ games/
│ ├ index.md # ゲーム一覧
│ └ sweetlands.md # 個別ゲームページ
├ assets/
│ └ thumbs/ # サムネイル画像
└ _layouts/
└ default.html # 共通レイアウト

## ページ構成
- index.md（全体トップ）
- games/index.md（ゲーム一覧）
- games/xxx.md（個別ゲーム）

※ 個別ゲームはすべて games 配下に配置する

## ゲーム個別ページの考え方
- 1ゲームにつき1Markdownファイルとする
- 各ゲームページの中に以下を含める
  - ルールブックPDFへのリンク
  - ルール要点（要約）
  - 勘違いやすい点・気づき
  - 
- 個別ゲームページは **すべて `games/` 配下に配置**
- フォルダ分けは最小限とし、分類はメタ情報で管理する

---

## 5. ゲーム個別ページ仕様

### 5.1 基本方針
- 1ゲーム = 1Markdownファイル
- 表示構成は **レイアウトで統一**
- ページ本文には見出し `# タイトル` を書かない

### 5.2 front matter（メタ情報）
各ゲームページは以下の情報を持つ：

```yaml
layout: default
title: ゲーム名
players_min: 数値
players_max: 数値
time_min: 分
time_max: 分
weight: light | medium | heavy
age: 対象年齢
tags:
  - ジャンル
  - メカニクス
rulebook_url: 外部PDFリンク
thumbnail_url: サムネイル画像URL
thumbnail_alt: 画像代替テキスト
bgg_url: BGGページURL（任意）
※ 分類・検索・一覧生成は、すべてこのメタ情報を元に行う想定

6. ゲーム個別ページの表示構成
表示順は以下で統一する：

ページタイトル（レイアウト側で表示）

サムネイル画像（BGG風サイズ）

本文コンテンツ

本文に含める内容：

ルールブックPDFへのリンク

ルール要点（要約）

勘違いやすい点

補足・気づき・Q&A

7. ゲーム一覧ページ仕様
games/index.md に集約

将来的に以下の表示を想定

サムネイル付きカード表示

プレイ人数・時間・重さの簡易表示

初期段階では静的リンク一覧でも可

8. 分類・検索方針
8.1 分類軸
プレイ人数

プレイ時間

重さ（light / medium / heavy）

ジャンル・メカニクス（複数指定可）

8.2 管理方法
フォルダ分けは行わない

ゲームページのメタ情報で一元管理する

将来的に以下へ拡張可能な設計とする

分類別一覧ページ

JavaScriptによる簡易検索・フィルタ

9. ナビゲーション設計
PC表示

左サイドバーにナビゲーションを表示

モバイル表示

上部トップバーに簡易ナビゲーションを表示

トップ → ゲーム一覧 → 個別ゲーム の導線を重視

10. ルールPDFの扱い
ルールPDFは GitHub Pages には置かない

外部ストレージ（Dropbox 等）を利用

個別ゲームページからリンクして参照する

サイト側では表示・埋め込みは行わない

11. 運用方針
新規ゲーム追加時の作業は以下のみ

games/xxx.md を1ファイル追加

front matter を記入

サムネイルを assets または外部URLで指定

既存ページや一覧の大規模修正が不要な設計を目指す

初期は手動運用、将来的な自動化に備える

12. 非目標（やらないこと）
動的バックエンドの導入

会員管理・権限管理

ルールPDFの全文転載

初期段階での検索機能実装
